# PRD 交互式对话构建器

通过多轮引导对话（14个问题分4个阶段），探测用户需求，逐模块确认后输出完整 PRD。

## 技能激活

当用户说以下任意一句时激活本技能：
- "开始对话式 PRD"
- "用引导模式生成 PRD"
- "我要交互式地创建 PRD"
- "PRD 对话构建器"
- "对话式产品需求"

激活后：
1. 初始化状态文件 `tmp/prd-conversational/state.yaml`
2. 设置 `对话状态.是否激活 = true`
3. 从 Q0 开始对话
4. 每个问题回答后更新状态文件
5. 所有问题回答完后进入 PRD 生成
6. 生成后进入确认机制
7. 确认后保存 PRD 到 `docs/prd/`
8. 清除状态文件（或保留用于后续修改）

## 对话流程

### 阶段0: 意图理解（1个问题）

**进度**: [阶段0/4] 初始化

---

**系统消息**:
> 👋 你好！我是 PRD 对话构建器。
> 我会通过几个简单的问题，了解你想做什么产品，然后生成一份完整的需求文档。
>
> 请用一句话描述你想做的产品（例如：`"我想做一个 macOS 菜单栏语音输入 App"` 或 `"我要做一个团队代码片段分享工具"`）。

---

**Q0 - 初始想法**:
> 请描述你想做的产品：

**后处理**: 收到用户输入后，先用方案A的意图识别逻辑推断：
- 目标平台（macOS/iOS/Android/Web/CLI/跨平台）
- 核心功能类型（语音/图像/网络/数据持久化/系统集成）
- 交互模式（键盘/触摸/语音/手势/全局快捷键）

然后展示推断结果并请求确认：

```
✅ 我理解你想做一个 **[推断的平台]** **[推断的功能类型]**。

请确认或补充：
A. ✅ 确认（继续）
B. 修改为：[你的补充]
```

---

### 阶段1: 平台与基础（4个问题）

**进度**: [阶段1/4] 平台与基础 — 问题 1/4

---

**Q1 - 目标平台**:
> **目标平台是哪个？（可多选）**
>
> A. macOS 桌面应用
> B. iOS App（iPhone/iPad）
> C. Android App
> D. Web 应用
> E. 跨平台（Flutter / React Native / Electron）
> F. 后端服务 / API
> G. 命令行工具 / CLI
> H. Chrome Extension / 浏览器插件

**推荐**: 基于阶段0的推断，默认选中最可能的平台。

**后处理**: 记录回答 → 加载对应平台的知识库 → 推断相关权限和API需求。

---

**Q2 - 最低版本要求**:
> **需要支持的最低系统版本是？**
>
> A. 最新版本（推荐）
> B. 最近1-2个版本
> C. 最近3-4个版本
> D. 指定版本：______

**后处理**: 记录版本要求 → 影响 UI 框架和 API 的选择。

---

**Q3 - 目标用户**:
> **主要目标用户是？**
>
> A. 通用消费者
> B. 专业人士（如设计师/开发者/数据分析师）
> C. 企业用户（B2B）
> D. 内部团队使用

**后处理**: 记录目标用户 → 影响 UI 设计、安全要求、付费模式。

---

**Q4 - MVP vs 完整版**:
> **这次想实现的是？**
>
> A. MVP（核心功能可用即可）
> B. 完整版本（包含所有计划功能）
> C. 演示版（展示核心概念）

**后处理**: 记录范围 → 影响功能模块的优先级排序。
